# Contributing

Thanks for your interest in Confluence Git Sync!

## Development setup

```bash
git clone https://github.com/DIAL-Studio/obsidian-confluence-git-sync.git
cd obsidian-confluence-git-sync
npm install
```

## Testing

```bash
npm test
```

## Pull request process

1. Create a feature branch from `main`.
2. Make your changes.
3. Add or update tests as needed.
4. Ensure all tests pass.
5. Submit a PR with a clear description of the change and why it's needed.

## Code style

- TypeScript with strict null checks
- 2 spaces for indentation
- Semicolons required
- Single quotes preferred

## Commit messages

Use conventional commits: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`.
